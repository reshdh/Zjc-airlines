# ZJC航空公司预订系统 - 接口连接指南

## 📋 接口分类说明

### ✅ **新增接口（必须连接）**

这些是新增的座位选择相关接口，**必须连接**才能使用座位选择功能：

#### 1. 获取可用座位列
- **接口**: `GET /api/seats/available`
- **参数**: 
  - `flightId` (必填): 航班ID
  - `cabinClass` (必填): 舱位等级 (ECONOMY/BUSINESS/FIRST)
- **用途**: 在订票页面显示可选的座位列（如 A、B、C、D）
- **返回**: 可用列列表和每列的可用座位数量

#### 2. 选择座位
- **接口**: `POST /api/seats/select`
- **请求体**:
```json
{
  "flightId": 1,
  "cabinClass": "ECONOMY",
  "ticketCount": 2,
  "selectedColumns": ["A", "B"]
}
```
- **用途**: 根据用户选择的列号分配具体座位（系统自动分配排号）
- **返回**: 分配的座位号列表，如果选择的列已满会返回警告信息

---

### 🔄 **需要更新的接口（参数有变化）**

这些接口的参数发生了变化，**需要更新**请求体：

#### 1. 创建预订（已支持座位选择）
- **接口**: `POST /api/bookings`
- **新增参数**: `selectedSeatColumns` (可选)
- **更新后的请求体**:
```json
{
  "flightId": 1,
  "userId": 1,
  "cabinClass": "ECONOMY",
  "ticketCount": 2,
  "payLater": false,
  "selectedSeatColumns": ["A", "B"]  // ⚠️ 新增：座位列选择（可选）
}
```
- **说明**: 
  - `selectedSeatColumns` 是**可选参数**，不传也可以正常订票（系统随机分配）
  - 如果传入，系统会根据选择的列分配座位
  - 如果选择的列已满，系统会自动分配其他列的座位并返回警告

## 📝 前端集成建议

### 订票流程（带座位选择）

```typescript
// 1. 用户选择航班和舱位后，获取可用座位列
const seatColumns = await seatApi.getAvailableSeatColumns(flightId, cabinClass);
// 返回: { availableColumns: ["A", "B", "C", "D"], columnAvailability: [...] }

// 2. 用户选择座位列（如选择 A、B）
const selectedColumns = ["A", "B"];

// 3. 创建预订时传入座位选择
const booking = await bookingApi.createBooking({
  flightId,
  userId,
  cabinClass,
  ticketCount: 2,
  selectedSeatColumns: selectedColumns  // ⚠️ 新增参数
});
```

### 订票流程（不带座位选择）

```typescript
// 不传 selectedSeatColumns，系统会随机分配座位
const booking = await bookingApi.createBooking({
  flightId,
  userId,
  cabinClass,
  ticketCount: 2
  // 不传 selectedSeatColumns
});
```

---

## ⚠️ 注意事项

1. **向后兼容**: `selectedSeatColumns` 是可选参数，不传也能正常订票
2. **座位分配逻辑**: 
   - 如果选择的列有可用座位，优先分配选择的列
   - 如果选择的列已满，系统会自动分配其他列的座位
   - 系统会自动分配排号，用户只需选择列号
3. **错误处理**: 如果座位选择失败，预订仍然可以创建（系统会随机分配）
