## 1. 系统在测试过程中遇到了哪些问题？如何解决的？

### 1.1 常见问题

#### 问题 1：事务回滚错误

**问题描述：**
- "Transaction silently rolled back" 错误
- 创建航班时出现

**原因：**
- 座位数据与现有订单冲突
- 事务中操作失败导致回滚

**解决方案：**
```java
private void syncSeatSystemFromLegacySeats(Flight flight) {
    // 检查是否有已售座位
    long activeSeatBindings = bookingSeatRepository.countActiveSeatBindings(flight.getId());
    if (activeSeatBindings > 0) {
        // 如果有已售座位，跳过自动清理
        return;
    }
    // 否则继续生成新座位数据
}
```

#### 问题 2：端口占用

**问题描述：**
- 端口 8080 被占用
- 应用无法启动

**解决方案：**
```bash
# Windows
netstat -ano | findstr :8080
taskkill /F /PID <PID>

# Linux/Mac
lsof -i :8080
kill -9 <PID>
```

#### 问题 3：数据库连接失败

**问题描述：**
- 数据库连接超时
- 连接池耗尽

**解决方案：**
- 检查数据库配置
- 增加连接池大小
- 检查网络连接

## 2. 如何部署和运行该系统？

### 2.1 开发环境部署

**后端启动：**
```bash
# 1. 安装 JDK 17 和 Maven
# 2. 配置数据库
# 3. 编译项目
mvn clean install

# 4. 启动应用
mvn spring-boot:run
# 或
java -jar target/zjc-airline-booking-backend-0.0.1-SNAPSHOT.jar
```

**前端启动：**
```bash
# 1. 安装 Node.js 18+
# 2. 安装依赖
cd zjc-airline-booking-frontend
npm install

# 3. 启动开发服务器
npm run dev
```

### 2.2 生产环境部署

**后端部署：**
```bash
# 1. 打包
mvn clean package

# 2. 运行 JAR
java -jar target/zjc-airline-booking-backend-0.0.1-SNAPSHOT.jar

# 3. 使用 systemd 管理（Linux）
sudo systemctl start zjc-airline-booking
```

**前端部署：**
```bash
# 1. 构建生产版本
npm run build

# 2. 启动生产服务器
npm start

# 3. 或部署到 Vercel/Netlify
vercel deploy
```

### 2.3 Docker 部署（未来）

**Dockerfile：**
