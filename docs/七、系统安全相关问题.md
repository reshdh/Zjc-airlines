## 1. 如何防止 SQL 注入攻击？

### 1.1 JPA 参数化查询

**使用 JPA Repository：**
```java
public interface FlightRepository extends JpaRepository<Flight, Long> {
    // 方法名自动生成查询（参数化）
    Optional<Flight> findByFlightNumber(String flightNumber);
}
```

**自动生成的 SQL（参数化）：**
```sql
SELECT * FROM flights WHERE flight_number = ?  -- ? 是参数占位符
```

**使用 @Query 注解：**
```java
@Query("SELECT f FROM Flight f WHERE f.origin = :origin AND f.destination = :destination")
List<Flight> findByOriginAndDestination(
    @Param("origin") String origin, 
    @Param("destination") String destination
);
```

### 1.2 避免拼接 SQL

**错误示例（容易 SQL 注入）：**
```java
// ❌ 错误：拼接 SQL（存在 SQL 注入风险）
String sql = "SELECT * FROM flights WHERE origin = '" + userInput + "'";
```

**正确示例（参数化查询）：**
```java
// ✅ 正确：使用参数化查询
@Query("SELECT f FROM Flight f WHERE f.origin = :origin")
List<Flight> findByOrigin(@Param("origin") String origin);
```

### 1.3 Spring Data JPA 的优势

**自动参数化：**
- Spring Data JPA 自动将方法参数转换为参数化查询
- 使用 PreparedStatement，自动转义特殊字符
- 防止 SQL 注入攻击

```typescript

// 如果需要显示 HTML（不推荐），使用 dangerouslySetInnerHTML
```

**输入验证：**
```typescript
// 验证和清理用户输入
function sanitizeInput(input: string): string {

    .replace(/</g, "&lt;")
    .replace(/>/g, "&gt;")
    .replace(/"/g, "&quot;")
    .replace(/'/g, "&#x27;");
}
```

## 3. 如何实现用户身份认证和授权？

### 3.1 密码加密

**BCrypt 加密：**
```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
}
```

**注册时加密：**
```java
public User register(RegisterRequest request) {
    User user = new User();
    user.setPassword(passwordEncoder.encode(request.getPassword())); // BCrypt 加密
    return userRepository.save(user);
}
```

**登录时验证：**
```java
public User login(LoginRequest request) {
    User user = userRepository.findByUsername(request.getUsername())
        .orElseThrow(() -> new RuntimeException("用户名或密码错误"));
    
    // BCrypt 密码匹配
    if (!passwordEncoder.matches(request.getPassword(), user.getPassword())) {
        throw new RuntimeException("用户名或密码错误");
    }
    
    return user;
}
```

### 3.2 会话管理

**前端存储：**
```typescript
// 登录成功后保存用户信息到 LocalStorage
```

## 4. 如何保护 API 接口的安全？

### 4.1 CORS 配置

**允许的源：**
```java
@Configuration
public class CorsConfig {
    @Bean
    public CorsConfigurationSource corsConfigurationSource() {
        CorsConfiguration configuration = new CorsConfiguration();
        configuration.setAllowedOrigins(Arrays.asList("http://localhost:3000"));
        configuration.setAllowedMethods(Arrays.asList("GET", "POST", "PUT", "DELETE"));
        configuration.setAllowedHeaders(Arrays.asList("*"));
        configuration.setAllowCredentials(true);
        
        UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
        source.registerCorsConfiguration("/api/**", configuration);
        return source;
    }
}
```

## 5. 如何处理敏感数据（如用户身份证号）的存储和传输？

```java
@Entity
public class User {

```

**加密转换器：**
```java
@Converter
public class EncryptedStringConverter implements AttributeConverter<String, String> {
    @Override
    public String convertToDatabaseColumn(String attribute) {
        // 加密
        return encrypt(attribute);
    }
    
    @Override
    public String convertToEntityAttribute(String dbData) {
        // 解密
        return decrypt(dbData);
    }
}
```

### 5.3 数据脱敏

**返回数据脱敏：**
```java
public class UserDTO {
    private String idNumber;
    
    // 只返回部分信息（脱敏）
    public String getIdNumberMasked() {
        if (idNumber == null || idNumber.length() < 8) {
            return idNumber;
        }
        // 保留前3位和后4位，中间用*代替
        return idNumber.substring(0, 3) + "****" + idNumber.substring(idNumber.length() - 4);
    }
}
```
