## 1. 航班信息展示功能是如何实现的？支持哪些查询条件？

### 1.1 功能概述

航班信息展示功能允许用户通过 Web 页面查看所有可用航班信息，支持多条件搜索和筛选。

### 1.2 后端实现

**Controller 层：**
```java
@RestController
@RequestMapping("/api/flights")
public class FlightController {
    
    @GetMapping
    public ResponseEntity<ApiResponse<List<Flight>>> getAllFlights(
            @RequestParam(defaultValue = "false") boolean includePast) {
        List<Flight> flights = flightService.getAllFlights(includePast);
        return ResponseEntity.ok(ApiResponse.success(flights));
    }
    
    @PostMapping("/search")
    public ResponseEntity<ApiResponse<List<Flight>>> searchFlights(
            @RequestBody FlightSearchRequest request,
            @RequestParam(defaultValue = "false") boolean includePast) {
        List<Flight> flights = flightService.searchFlights(request, includePast);
        return ResponseEntity.ok(ApiResponse.success(flights));
    }
}
```

**Service 层：**
```java
@Service
public class FlightService {
    
    public List<Flight> searchFlights(FlightSearchRequest request, boolean includePast) {
        String originKeyword = normalizeKeyword(request.getOrigin());
        String destinationKeyword = normalizeKeyword(request.getDestination());
        
        List<Flight> flights;
        if (originKeyword != null && destinationKeyword != null) {
            // 同时搜索出发地和目的地
            flights = flightRepository.findByOriginContainingIgnoreCaseAndDestinationContainingIgnoreCase(
                    originKeyword, destinationKeyword);
        } else if (originKeyword != null) {
            // 只搜索出发地
            flights = flightRepository.findByOriginContainingIgnoreCase(originKeyword);
        } else if (destinationKeyword != null) {
            // 只搜索目的地
            flights = flightRepository.findByDestinationContainingIgnoreCase(destinationKeyword);
        } else {
            // 无搜索条件，返回所有可用航班
            flights = flightRepository.findAvailableFlights();
        }
        return prepareFlightList(flights, includePast);
    }
}
```

### 1.3 支持的查询条件

**1. 出发地（Origin）**
- 支持中文搜索（如："北京"）
- 支持拼音搜索（如："beijing"、"bj"）
- 支持部分拼音搜索（如："beij"）
- 使用 `LIKE` 模糊查询，忽略大小写

**2. 目的地（Destination）**
- 支持中文搜索
- 支持拼音搜索
- 使用 `LIKE` 模糊查询，忽略大小写

**3. 出发日期（可选）**
- 前端支持日期选择器

**4. 是否包含历史航班（includePast）**
- `includePast=false`：只显示未起飞的航班
- `includePast=true`：显示所有航班（包括已起飞的）

## 2. 用户注册和登录的具体流程是什么？如何保证密码安全？

### 2.1 用户注册流程

#### 2.1.1 注册步骤

**前端步骤：**
1. 用户填写注册表单（用户名、密码、确认密码、姓名、身份证号、联系电话、地址）
2. 前端进行基本验证（必填项、密码长度、密码一致性等）
3. 提交注册请求到后端 API

**后端步骤：**
1. **Controller 层接收请求**（`UserController.register`）
2. **Service 层处理注册逻辑**（`UserService.register`）：
   - 检查用户名是否已存在
   - 检查身份证号是否已存在
   - 检查手机号是否已存在
   - 创建新用户对象
   - **密码加密**（使用 `BCryptPasswordEncoder`）
   - 设置默认角色为 "USER"
   - 设置账号状态为启用（status=1）
   - 初始化钱包余额（默认200元）
   - 保存用户到数据库
3. 返回注册结果

#### 2.1.2 密码安全保证

**密码加密：**
```java
@Service
public class UserService {
    
    @Autowired
    private PasswordEncoder passwordEncoder;
    
    public User register(RegisterRequest request) {
        // ... 验证逻辑 ...
        
        User user = new User();
        user.setPassword(passwordEncoder.encode(request.getPassword())); // BCrypt 加密
        // ... 其他设置 ...
        
        return userRepository.save(user);
    }
}
```

**BCrypt 密码加密优势：**
- **不可逆加密：** 无法从加密后的密码反推出原始密码
- **盐值（Salt）自动生成：** 每次加密都会生成随机盐值，即使相同密码，加密结果也不同
- **自适应成本：** 可以通过调整成本因子（rounds）来平衡安全性和性能
- **行业标准：** BCrypt 是业界广泛使用的密码加密算法

**配置方式：**
```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    
    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder(); // 默认强度为 10
    }
}
```

### 2.2 用户登录流程

#### 2.2.1 登录步骤

**前端步骤：**
1. 用户输入用户名和密码
2. 提交登录请求到后端 API

**后端步骤：**
1. **Controller 层接收请求**（`UserController.login`）
2. **Service 层验证登录**（`UserService.login`）：
   - 根据用户名查找用户
   - 如果用户不存在，返回"用户名或密码错误"
   - 使用 `PasswordEncoder.matches()` 验证密码
   - 如果密码不匹配，返回"用户名或密码错误"
   - 初始化钱包余额（如果为空）
   - 返回用户信息（不包含密码）
3. **前端处理响应**：
   - 将用户信息保存到 `LocalStorage`
   - 根据用户角色跳转到相应页面（普通用户→首页，管理员→管理后台）

#### 2.2.2 密码验证

**密码匹配：**
```java
public User login(LoginRequest request) {
    Optional<User> userOpt = userRepository.findByUsername(request.getUsername());
    if (userOpt.isEmpty()) {
        throw new RuntimeException("用户名或密码错误");
    }
    
    User user = userOpt.get();
    // BCrypt 密码匹配（会自动处理盐值）
    if (!passwordEncoder.matches(request.getPassword(), user.getPassword())) {
        throw new RuntimeException("用户名或密码错误");
    }
    
    return user;
}
```

**安全措施：**
- **统一错误提示：** 无论用户名不存在还是密码错误，都返回"用户名或密码错误"，防止用户枚举攻击
- **密码不返回前端：** 用户对象中不包含密码字段，避免密码泄露

## 3. 机票预订功能的完整流程是什么？包括哪些关键步骤？

### 3.1 预订流程概述

机票预订是一个复杂的业务流程，涉及多个步骤和验证。

### 3.2 完整流程步骤

#### 步骤 1：用户选择航班和舱位

**前端：**
- 用户在航班列表页面点击"预订"按钮
- 选择舱位等级（经济舱/商务舱/头等舱）
- 选择订票数量（最多9人）

#### 步骤 2：填写乘客信息

**前端：**
- 显示乘客信息表单（根据订票数量动态生成）
- 每位乘客需要填写：姓名、身份证号、联系电话
- 第一位乘客默认使用当前登录用户的信息

**后端验证：**
```java
private void validatePassengerIdentities(BookingRequest request) {
    Set<String> idNumbers = new HashSet<>();
    Set<String> phones = new HashSet<>();
    
    for (BookingRequest.PassengerInfo passenger : request.getPassengers()) {
        String idNumber = passenger.getIdNumber();
        String phone = passenger.getPhone();
        
        // 验证身份证号唯一性
        if (!idNumbers.add(idNumber)) {
            throw new RuntimeException("身份证号 " + idNumber + " 已被重复使用");
        }
        
        // 验证联系电话唯一性
        if (!phones.add(phone)) {
            throw new RuntimeException("联系电话 " + phone + " 已被重复使用");
        }
    }
}
```

#### 步骤 3：验证座位数量

**后端：**
```java
// 获取剩余座位数
int remainingSeats = calculateRemainingSeatsFromSeatTable(flightId, cabinClass);

// 验证订票数不超过剩余座位数
if (remainingSeats < request.getTicketCount()) {
    throw new RuntimeException("该舱位剩余座位不足");
}
```

#### 步骤 4：计算订单金额

**价格计算：**
```java
// 1. 计算基础价格
BigDecimal baseAmount = cabinPrice.multiply(BigDecimal.valueOf(ticketCount));

// 2. 判断是否在紧急窗口（起飞前12小时内）
boolean urgentDeparture = isWithinUrgentWindow(flight.getDepartureTime());

// 3. 如果紧急，加收10%紧急加价
BigDecimal surchargeRate = urgentDeparture ? URGENT_SURCHARGE_RATE : BigDecimal.ZERO;
BigDecimal totalAmount = baseAmount.add(baseAmount.multiply(surchargeRate))
        .setScale(2, RoundingMode.HALF_UP);
```

#### 步骤 5：验证钱包余额

**余额检查：**
```java
BigDecimal walletBalance = user.getWalletBalance();

// 立即支付需要检查余额
if (!payLater && walletBalance.compareTo(totalAmount) < 0) {
    throw new RuntimeException("钱包余额不足，请先充值或减少购票数量");
}
```

#### 步骤 6：创建订单

**订单创建：**
```java
Booking booking = new Booking();
booking.setFlight(flight);
booking.setUser(user);
booking.setCabinClass(request.getCabinClass());
booking.setTicketCount(request.getTicketCount());
booking.setTotalAmount(totalAmount);
booking.setUrgentSurchargeRate(surchargeRate);
booking.setStatus(payLater ? "CREATED" : "PAID");
booking.setPaymentDueAt(payLater ? LocalDateTime.now().plusMinutes(15) : null);

// 订单号自动生成（@PrePersist 回调）
Booking savedBooking = bookingRepository.save(booking);
```

#### 步骤 7：分配座位

**座位分配：**
```java
SeatSelectionRequest seatRequest = new SeatSelectionRequest();
seatRequest.setFlightId(request.getFlightId());
seatRequest.setCabinClass(request.getCabinClass());
seatRequest.setTicketCount(request.getTicketCount());
seatRequest.setSelectedColumns(request.getSelectedSeatColumns());

SeatSelectionResponse seatResponse = seatService.selectSeats(seatRequest);

// 将分配的座位与订单关联
for (int i = 0; i < assignedSeats.size(); i++) {
    BookingSeat bookingSeat = new BookingSeat();
    bookingSeat.setBooking(savedBooking);
    bookingSeat.setSeatNumber(assignedSeats.get(i));
    bookingSeat.setPassengerName(passengerNames.get(i));
    bookingSeatRepository.save(bookingSeat);
}
```

#### 步骤 8：扣减钱包余额

**立即支付：**
```java
if (!payLater) {
    user.setWalletBalance(walletBalance.subtract(totalAmount));
    userRepository.save(user);
}
```

**稍后支付：**
- 设置 `paymentDueAt` 为当前时间 + 15分钟
- 15分钟后自动取消订单（通过定时任务）

#### 步骤 9：更新剩余座位数

**旧系统（flight_seats）：**
```java
if (!useNewSeatSystem && legacyFlightSeat != null) {
    legacyFlightSeat.setRemainingSeats(
        legacyFlightSeat.getRemainingSeats() - request.getTicketCount());
    flightSeatRepository.save(legacyFlightSeat);
}
```

**新系统（seats + booking_seats）：**
- 通过 `booking_seats` 表记录已占用的座位
- 剩余座位数 = 总座位数 - 已占用座位数（实时计算）

### 3.3 关键步骤总结

1. ✅ **用户选择航班和舱位**
2. ✅ **填写乘客信息（验证唯一性）**
3. ✅ **验证座位数量**
4. ✅ **计算订单金额（含紧急加价）**
5. ✅ **验证钱包余额**
6. ✅ **创建订单（自动生成订单号）**
7. ✅ **分配座位（避免重复分配）**
8. ✅ **扣减钱包余额（立即支付）**
9. ✅ **更新剩余座位数**

## 4. 座位选择功能如何实现？如何确保座位不被重复选择？

### 4.1 座位系统架构

系统采用**双座位系统**以支持数据迁移：

**旧系统（Legacy）：**
- `flight_seats` 表：按舱位统计总座位数和剩余座位数
- 座位信息存储在航班实体的 `seats` 字段中

**新系统（Current）：**
- `seats` 表：每个座位的详细信息（座位号、列号、排号、舱位、是否可用）
- `booking_seats` 表：订单与座位的关联表（记录哪些座位被哪些订单占用）

### 4.2 座位选择流程

#### 4.2.1 获取可用座位列

**API 端点：** `GET /api/seats/available/{flightId}?cabinClass=ECONOMY`

**实现逻辑：**
```java
public SeatSelectionResponse getAvailableSeatColumns(Long flightId, String cabinClass) {
    // 1. 获取所有座位
    List<Seat> allSeats = seatRepository.findByFlightIdAndCabinClass(flightId, cabinClass);
    
    // 2. 获取已占用的座位ID
    Set<Long> occupiedSeatIds = loadOccupiedSeatIds(flightId, cabinClass);
    
    // 3. 统计每列的可用座位数
    Map<String, Long> availableCountMap = new HashMap<>();
    for (Seat seat : allSeats) {
        String column = seat.getSeatColumn();
        boolean seatAvailable = seat.getIsAvailable() 
            && !occupiedSeatIds.contains(seat.getId());
        if (seatAvailable) {
            availableCountMap.put(column, availableCountMap.getOrDefault(column, 0L) + 1);
        }
    }
    
    return response;
}
```

#### 4.2.2 选择座位

**API 端点：** `POST /api/seats/select`

**实现逻辑：**
```java
@Transactional
public SeatSelectionResponse selectSeats(SeatSelectionRequest request) {
    // 1. 加载已占用的座位ID集合
    Set<Long> occupiedSeatIds = loadOccupiedSeatIds(request.getFlightId(), cabinClass);
    
    // 2. 验证每个选择的列是否有可用座位
    for (String column : selectedColumns) {
        List<Seat> columnSeats = seatRepository.findAvailableSeatsByColumn(
            request.getFlightId(), cabinClass, column);
        long availableCount = columnSeats.stream()
            .filter(seat -> !occupiedSeatIds.contains(seat.getId()))
            .count();
        if (availableCount == 0) {
            unavailableColumns.add(column); // 记录已满的列
        }
    }
    
    // 3. 为每个列分配一个座位（取第一个可用且未被占用的座位）
    for (int i = 0; i < ticketCount; i++) {
        String column = columnsToAssign.get(i);
        List<Seat> columnSeats = seatRepository.findAvailableSeatsByColumn(
            request.getFlightId(), cabinClass, column);
        
        Optional<Seat> availableSeat = columnSeats.stream()
            .filter(seat -> !occupiedSeatIds.contains(seat.getId())) // 过滤已占用
            .findFirst();
        
        if (availableSeat.isPresent()) {
            Seat seat = availableSeat.get();
            assignedSeats.add(seat.getSeatNumber());
            occupiedSeatIds.add(seat.getId()); // 标记为已占用（本次分配）
        }
    }
    
    return response;
}
```

### 4.3 防止重复选择的机制

#### 4.3.1 数据库层面

**唯一性约束：**
- `booking_seats` 表中，`seat_id` 和 `booking_id` 的组合确保一个座位只能被一个订单占用
- 数据库外键约束保证数据完整性

#### 4.3.2 应用层面

**1. 事务隔离（@Transactional）**
```java
@Transactional
public SeatSelectionResponse selectSeats(SeatSelectionRequest request) {
    // 在事务中执行，保证原子性
}
```

**2. 实时检查占用状态**
```java
// 每次分配前检查座位是否已被占用
Set<Long> occupiedSeatIds = loadOccupiedSeatIds(flightId, cabinClass);

// 过滤已占用的座位
.filter(seat -> !occupiedSeatIds.contains(seat.getId()))
```

**3. 内存标记（同一次分配内）**
```java
// 在同一次分配中，已分配的座位ID会被添加到 occupiedSeatIds 集合中
// 避免同一次请求中重复分配同一个座位

```

#### 4.3.3 并发控制

**乐观锁（未来优化）：**

**1. 按列分配**

- 系统在每个选择的列中分配一个座位

**2. 自动补充**

## 5. 订单管理功能包括哪些操作？如何实现订单的修改、增加和退票？

### 5.1 订单管理功能列表

**用户端操作：**
1. **查看订单列表** - 查看自己的所有订单
2. **查看订单详情** - 查看订单的详细信息（航班、乘客、座位等）
3. **支付订单** - 支付待支付订单（15分钟内）
4. **取消订单** - 取消待支付订单
5. **申请退票** - 申请退票（已支付订单）
6. **申请改签** - 申请改签（已支付订单）

**管理员端操作：**
1. **查看所有订单** - 查看系统中所有订单
2. **审核退票申请** - 同意或驳回退票申请
3. **订单统计** - 查看订单统计数据

### 5.2 订单状态

**状态定义：**
- **CREATED：** 已创建但未支付（需在15分钟内完成支付）
- **PAID：** 已支付并出票
- **REFUND_REVIEW：** 退票申请审核中
- **REFUND_REJECTED：** 退票申请已驳回
- **CANCELED：** 已取消（超时未支付、用户主动取消、退票成功）

### 5.3 订单修改（改签）

**改签流程：**
1. 用户选择要改签的订单（必须是 PAID 状态）
2. 选择新航班和舱位（必须与原订单不同）
3. 系统计算改签费用和差价
4. 用户确认并支付差价（或接收退款）
5. 原订单标记为已改签，创建新订单

**实现代码：**
```java
public Booking changeBooking(Long bookingId, ChangeBookingRequest request) {
    Booking originalBooking = bookingRepository.findById(bookingId)
        .orElseThrow(() -> new RuntimeException("订单不存在"));
    
    // 验证订单状态
    if (!"PAID".equals(originalBooking.getStatus())) {
        throw new RuntimeException("只有已支付的订单可以改签");
    }
    
    // 验证改签时间（必须在起飞前2小时）
    LocalDateTime now = LocalDateTime.now();
    if (now.isAfter(originalBooking.getFlight().getDepartureTime().minusHours(2))) {
        throw new RuntimeException("改签必须在航班起飞前2小时进行");
    }
    
    // 验证不能改签到同一航班的相同舱位
    Flight newFlight = flightRepository.findById(request.getNewFlightId())
        .orElseThrow(() -> new RuntimeException("新航班不存在"));
    
    if (Objects.equals(newFlight.getId(), originalBooking.getFlight().getId())
        && request.getNewCabinClass().equalsIgnoreCase(originalBooking.getCabinClass())) {
        throw new RuntimeException("不能改签到同一航班的相同舱位");
    }
    
    // 计算改签费用和差价
    ChangePriceResponse priceResponse = calculateChangePrice(bookingId, request);
    
    // 创建新订单
    Booking newBooking = new Booking();
    // ... 设置新订单信息 ...
    newBooking.setOriginalBookingId(originalBooking.getId());
    newBooking.setChangeFeeAmount(priceResponse.getChangeFee());
    
    // 更新原订单状态
    originalBooking.setStatus("CANCELED");
    
    return bookingRepository.save(newBooking);
}
```

### 5.5 订单退票

**退票流程：**
1. 用户选择要退票的订单（必须是 PAID 状态）
2. 填写退票原因（可选）
3. 提交退票申请
4. 系统判断是否需要审核：
   - **需要审核：** 起飞前12小时内退票、24小时内取消订单3次及以上
   - **不需要审核：** 其他情况自动退款
5. 如果需要审核，订单状态变为 `REFUND_REVIEW`，等待管理员审核
6. 如果不需要审核，系统自动计算退款金额并退款

**实现代码：**
```java
public Booking refundBooking(Long bookingId, String reason) {
    Booking booking = bookingRepository.findById(bookingId)
        .orElseThrow(() -> new RuntimeException("订单不存在"));
    
    // 验证订单状态
    if (!"PAID".equals(booking.getStatus())) {
        throw new RuntimeException("只有已支付的订单可以退票");
    }
    
    // 判断是否需要审核
    boolean needsReview = shouldRequireRefundReview(booking);
    
    if (needsReview) {
        // 需要审核
        booking.setStatus("REFUND_REVIEW");
        booking.setRefundReason(reason);
    } else {
        // 自动退款
        processRefund(booking, reason);
    }
    
    return bookingRepository.save(booking);
}

private boolean shouldRequireRefundReview(Booking booking) {
    LocalDateTime now = LocalDateTime.now();
    LocalDateTime departureTime = booking.getFlight().getDepartureTime();
    
    // 起飞前12小时内退票需要审核
    if (now.isAfter(departureTime.minusHours(12))) {
        return true;
    }
    
    // 24小时内取消订单3次及以上需要审核
    long cancelCount = bookingRepository.countByUserIdAndStatusAndUpdatedAtAfter(
        booking.getUser().getId(), "CANCELED", now.minusDays(1));
    if (cancelCount >= 3) {
        return true;
    }
    
    return false;
}

private void processRefund(Booking booking, String reason) {
    // 计算退款金额
    BigDecimal refundAmount = calculateRefundAmount(booking);
    
    // 退款到用户钱包
    User user = booking.getUser();
    user.setWalletBalance(user.getWalletBalance().add(refundAmount));
    userRepository.save(user);
    
    // 释放座位
    releaseSeats(booking);
    
    // 更新订单状态
    booking.setStatus("CANCELED");
    booking.setRefundReason(reason);
}
```

**退款金额计算：**
```java
private BigDecimal calculateRefundAmount(Booking booking) {
    BigDecimal totalAmount = booking.getTotalAmount();
    LocalDateTime now = LocalDateTime.now();
    LocalDateTime departureTime = booking.getFlight().getDepartureTime();
    
    // 起飞前12小时内退票，扣除1%手续费
    if (now.isAfter(departureTime.minusHours(12))) {
        BigDecimal refundFeeRate = URGENT_REFUND_FEE_RATE; // 0.01 (1%)
        BigDecimal refundFee = totalAmount.multiply(refundFeeRate);
        booking.setRefundFeeRate(refundFeeRate);
        booking.setRefundFeeAmount(refundFee);
        return totalAmount.subtract(refundFee);
    } else {
        // 起飞前12小时外退票，全额退款
        booking.setRefundFeeRate(BigDecimal.ZERO);
        booking.setRefundFeeAmount(BigDecimal.ZERO);
        return totalAmount;
    }
}
```

## 7. 如何实现多用户同时访问和操作？

### 7.1 无状态设计

**当前实现：**
- 后端服务采用无状态设计，每个请求都是独立的
- 用户信息存储在客户端（LocalStorage），不存储在服务器会话中
- 每个请求都需要携带用户信息（通过请求体或请求头）

**优势：**
- 支持水平扩展（可以部署多个服务实例）
- 不依赖服务器会话存储，降低内存压力
- 便于负载均衡

### 7.2 数据库并发控制

**事务隔离级别：**
- MySQL InnoDB 默认使用 `REPEATABLE READ` 隔离级别
- 保证在同一事务中多次读取同一数据的一致性

**事务管理：**
