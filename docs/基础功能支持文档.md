## 1. 系统概述

ZJC 航空公司机票预订系统是一个基于 Web 的机票预订平台，支持用户在线查看航班信息、预订机票、管理订单，以及管理员维护航班信息等功能。

### 1.1 技术架构

- **后端：** Spring Boot 3.3.0 + Spring Data JPA + MySQL
- **前端：** Next.js 14 + React + TypeScript + Chakra UI
- **数据库：** MySQL 8.0+
- **安全：** Spring Security + BCrypt 密码加密

### 1.2 系统访问

- **前端地址：** `http://localhost:3000`
- **后端 API：** `http://localhost:8080`
- **数据库：** MySQL `zjc_airline_booking_db`

### 2.4 机票预订功能 ✅

**功能描述：** 在航班信息中选择要预定的航班号，选择订票数（订票数要小于剩余票数），自动生成总金额，提交后即生成订票单。

**实现状态：** ✅ 已实现

**预订流程：**
1. 选择航班和舱位等级（经济舱/商务舱/头等舱）
2. 选择订票数量（必须小于该舱位剩余座位数）
3. 填写乘客信息（支持多人订票）
4. 选择座位列（可选）
5. 系统自动计算总金额（含紧急加价）
6. 选择立即支付或稍后支付
7. 提交订单生成订票单

**功能特性：**
- 支持多人订票（每位乘客填写姓名、身份证号、联系电话）
- 自动验证乘客信息唯一性（身份证号和联系电话必须唯一，姓名可以相同）
- 自动计算订单总金额（含紧急加价）
- 支持立即支付或稍后支付（15分钟倒计时）
- 自动分配座位号（如选择座位列，按列分配；否则自动分配）
- 订票成功后自动扣减钱包余额

**页面路径：** `/flights`

## 3. 特殊要求功能

### 3.1 12小时内退票收手续费1% ✅

**功能描述：** 航班出发前12小时内退票，收取订单金额的1%作为退票手续费。

**实现状态：** ✅ 已实现

**退票手续费规则：**
- **起飞前12小时内退票：** 收取订单金额的 **1%** 作为退票手续费
- **起飞前12小时外退票：** 不收取退票手续费，全额退款

**退票审核机制：**
- 起飞前12小时内退票需要管理员审核
- 24小时内取消订单3次及以上需要管理员审核
- 其他情况自动退款，无需审核

**退款计算：**
```
退款金额 = 订单金额 - (订单金额 × 退票手续费率)
```

### 3.2 航班出发前12小时内票价上涨10% ✅

**功能描述：** 航班出发前12小时内订票，票价自动上浮10%作为紧急加价。

**实现状态：** ✅ 已实现

**紧急加价规则：**
- **起飞前12小时内订票：** 在原价基础上加收 **10%** 作为紧急加价
- **起飞前12小时外订票：** 不收取紧急加价，按原价计算

**价格计算：**
```
最终订单金额 = 基础价格 × (1 + 紧急加价率)
```

### 3.3 订票时可以添加多人信息 ✅

**功能描述：** 订票时可以添加多人信息进行订票。

**实现状态：** ✅ 已实现

**多人订票规则：**
- 支持一次预订为多人购票（最多9人）
- 每位乘客需要填写：
  - 姓名（可以相同，支持重名）
  - 身份证号（必须唯一，不能重复）
  - 联系电话（必须唯一，不能重复）
- 系统自动验证乘客信息唯一性
- 第一位乘客默认使用当前登录用户的信息
- 自动为每位乘客分配座位号

**验证规则：**
- 每位乘客的身份证号必须唯一
- 每位乘客的联系电话必须唯一
- 姓名可以相同（支持重名情况）

## 4. 退改签规则

### 4.1 退票规则

**退票条件：**
- 只有已支付（PAID）状态的订单可以申请退票
- 改签后的新订单不能退票
- 已改签的原订单不能退票

**退票手续费：**
- **起飞前12小时内退票：** 收取订单金额的 **1%** 作为退票手续费
- **起飞前12小时外退票：** 不收取退票手续费，全额退款

**退票流程：**
1. 用户选择要退票的订单
2. 填写退票原因（可选）
3. 提交退票申请
4. 系统判断是否需要审核：
   - 需要审核：订单进入审核状态，等待管理员处理
   - 不需要审核：系统自动计算退款金额并退款
5. 退款到用户钱包余额
6. 释放订单占用的座位
7. 订单状态更新为已取消

**退票审核：**
- 起飞前12小时内退票需要管理员审核
- 24小时内取消订单3次及以上需要管理员审核
- 管理员可以同意或驳回退票申请
- 驳回时必须填写驳回原因

### 4.2 改签规则

**改签条件：**
- 只有已支付（PAID）状态的订单可以申请改签
- 改签必须在原航班起飞前至少 **2小时** 进行
- 改签后的新订单不能再次改签
- 改签必须选择不同的航班或不同的舱位（不能改签到同一航班的相同舱位）
- 改签航班的出发地和目的地必须与原航班一致
- 新航班的起飞时间不能早于原航班

**改签手续费：**
- 改签手续费：新订单金额的 **5%**

**改签价格计算：**
1. **新订单金额计算：**
   - 基础价格 = 新航班舱位价格 × 票数
   - 如果新航班起飞前12小时内订票，加收10%紧急加价
   - 最终新订单金额 = 基础价格 × (1 + 紧急加价率)

2. **原订单退款计算：**
   - 如果原航班起飞前12小时内，扣除1%退票手续费
   - 如果原航班起飞前12小时外，全额退款

3. **差价计算：**
   ```
   差价 = 新订单金额 + 改签手续费 - 原订单退款
   ```
   - 差价为正值：用户需补缴差价
   - 差价为负值：系统退还差价到用户钱包

**改签流程：**
1. 用户选择要改签的订单
2. 选择新航班和舱位（必须与原订单不同）
3. 系统自动计算改签费用和差价
4. 用户确认并支付差价（或接收退款）
5. 原订单标记为已改签
6. 创建新订单，关联原订单
7. 为新订单分配新的座位号

- 支持钱包充值
- 订票时从钱包余额扣款
- 退票/改签时退款到钱包余额
- 支持查看钱包交易记录

### 7.4 座位管理系统 ✅

**功能描述：** 系统支持详细的座位管理。

**实现状态：** ✅ 已实现

**功能详情：**
- 支持三种舱位等级（经济舱、商务舱、头等舱）
- 自动生成座位布局（按机型配置）
- 支持按座位列选择座位
- 自动为乘客分配座位号
- 改签时自动分配新座位号

### 7.5 订单状态管理 ✅

**功能描述：** 系统支持完整的订单状态管理。

**实现状态：** ✅ 已实现

**订单状态：**
- **CREATED：** 已创建但未支付（需在15分钟内完成支付，否则自动取消）
- **PAID：** 已支付并出票
- **REFUND_REVIEW：** 退票申请审核中
- **REFUND_REJECTED：** 退票申请已驳回
- **CANCELED：** 已取消（超时未支付、用户主动取消、退票成功）

**自动处理：**
- 待支付订单超时15分钟自动取消
- 取消订单时自动释放座位
- 退票成功时自动释放座位
