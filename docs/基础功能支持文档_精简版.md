## 1. 系统概述

ZJC 航空公司机票预订系统是一个基于 Web 的机票预订平台，用于提高机票预订效率，降低工作的人力、物力开支，提高工作的准确性，方便旅客进行机票预订。

### 1.1 技术架构

- **后端：** Spring Boot 3.3.0 + Spring Data JPA + MySQL
- **前端：** Next.js 14 + React + TypeScript + Chakra UI
- **数据库：** MySQL 8.0+
- **安全：** Spring Security + BCrypt 密码加密

### 1.2 系统访问

- **前端地址：** `http://localhost:3000`
- **后端 API：** `http://localhost:8080`
- **数据库：** MySQL `zjc_airline_booking_db`

## 2. 基础功能要求

### 2.1 Web访问与航班信息查看 ✅

**项目要求：** 系统可以通过web访问,任何人可以查看首页上该公司所有的航班信息(航班号、机型、出发地、目的地、出发时间、到达时间、票价、剩余票数、备注)

**实现状态：** ✅ 已实现

**功能详情：**
- 首页展示所有航班信息
- 支持按出发地、目的地、日期搜索航班
- 显示完整的航班信息：
  - **航班号：** 每个航班的唯一标识
  - **机型：** 飞机型号（如：波音737-800）
  - **出发地：** 航班出发城市
  - **目的地：** 航班到达城市
  - **出发时间：** 航班起飞时间
  - **到达时间：** 航班降落时间
  - **票价：** 机票价格（显示最低价格）
  - **剩余票数：** 可预订的座位数量
  - **备注：** 航班相关信息说明

**页面路径：**
- 首页：`/`
- 航班搜索页：`/flights`

### 2.2 用户注册功能 ✅

**项目要求：** 旅客要订票,必须在系统中注册,填写注册信息表,包括姓名、身份证号、联系电话、地址

**实现状态：** ✅ 已实现

**注册信息要求：**
- **姓名：** 必填，旅客真实姓名
- **身份证号：** 必填，唯一，支持15位或18位身份证号
- **联系电话：** 必填，唯一，11位手机号
- **地址：** 必填，通讯地址（至少5个字符）
- **用户名：** 必填，唯一，至少3个字符（用于登录）
- **密码：** 必填，至少6个字符（用于登录）
- **确认密码：** 必填，需与密码一致

**功能特性：**
- 自动检查用户名、身份证号、手机号是否已存在
- 密码使用 BCrypt 加密存储，确保安全
- 注册成功后自动初始化钱包余额
- 注册成功后自动跳转到登录页面

**页面路径：** `/register`

### 2.3 用户登录功能 ✅

**项目要求：** 注册成功的顾客可以登录系统查看航班信息,进行订票操作

**实现状态：** ✅ 已实现

**登录要求：**
- **用户名：** 必填，注册时填写的用户名
- **密码：** 必填，注册时设置的密码

**功能特性：**
- 支持普通用户和管理员两种角色
- 登录成功后保存用户信息到本地存储
- 自动加载用户钱包余额
- 根据角色自动跳转到相应页面（普通用户→首页，管理员→管理后台）

**页面路径：** `/login`

### 2.4 机票预订功能 ✅

**项目要求：** 在航班信息中选择要预定的航班号,选择订票数(订票数要小于剩余票数),自动生成总金额,提交后即生成订票单

**实现状态：** ✅ 已实现

**预订流程：**
1. **选择航班：** 从航班列表中选择要预订的航班号
2. **选择舱位：** 选择舱位等级（经济舱/商务舱/头等舱）
3. **选择订票数：** 输入订票数量（必须小于该舱位剩余票数）
4. **填写乘客信息：** 填写每位乘客的姓名、身份证号、联系电话
5. **系统自动计算：** 系统根据票数和舱位价格自动生成总金额
6. **提交订单：** 确认信息后提交，系统自动生成订票单

**功能特性：**
- 自动验证订票数不超过剩余票数
- 自动计算订单总金额（基础价格 × 票数）
- 支持立即支付或稍后支付（15分钟倒计时）
- 订票成功后自动扣减钱包余额
- 自动分配座位号给每位乘客

**页面路径：** `/flights`

### 2.5 订单管理功能 ✅

**项目要求：** 顾客可以登录系统查看订票单,修改、增加或者退票

**实现状态：** ✅ 已实现

**订单管理功能：**
- **查看订票单：** 查看所有已创建的订票单列表
- **查看订单详情：** 查看订单详细信息（航班信息、乘客信息、座位信息、金额等）
- **修改订单：** 支持修改订单（改签功能，可更换航班或舱位）
- **增加订单：** 可以新增订单（再次预订其他航班）
- **退票：** 支持申请退票，系统根据退票时间计算退票手续费

**订单状态：**
- **CREATED：** 已创建但未支付（需在15分钟内完成支付，否则自动取消）
- **PAID：** 已支付并出票
- **REFUND_REVIEW：** 退票申请审核中
- **CANCELED：** 已取消（超时未支付、用户主动取消、退票成功）

**页面路径：** `/bookings`

### 2.6 管理员航班管理功能 ✅

**项目要求：** 航空公司订票系统管理员登录系统后,可以增加、修改、删除航班信息

**实现状态：** ✅ 已实现

**管理员功能：**
- **增加航班信息：**
  - 添加新航班的基本信息（航班号、机型、出发地、目的地、出发时间、到达时间、备注）
  - 配置各舱位的票价和座位数（经济舱、商务舱、头等舱）
  - 系统自动生成座位布局

- **修改航班信息：**
  - 修改航班基本信息（航班号除外，航班号唯一不可修改）
  - 修改舱位配置（票价、座位数）
  - 修改备注信息

- **删除航班信息：**
  - 删除不需要的航班
  - 系统会检查是否有关联订单，如果有关联订单则不允许删除
  - 删除航班会级联删除关联的舱位信息

**航班信息包括：**
- 航班号（唯一标识）
- 机型（如：波音737-800）
- 出发地
- 目的地
- 出发时间
- 到达时间
- 票价（各舱位价格）
- 剩余票数（各舱位剩余座位数）
- 备注信息

**页面路径：** `/admin`（仅管理员可访问）

## 3. 特殊要求功能

### 3.1 12小时内退票收手续费1% ✅

**项目要求：** 12小时内退票收手续费1%

**实现状态：** ✅ 已实现

**退票手续费规则：**
- **起飞前12小时内退票：** 收取订单金额的 **1%** 作为退票手续费
- **起飞前12小时外退票：** 不收取退票手续费，全额退款

**退款计算方式：**
```
退款金额 = 订单金额 - (订单金额 × 退票手续费率)
```

**退票审核机制：**
- 起飞前12小时内退票需要管理员审核
- 其他情况自动退款，无需审核
- 退款自动退回到用户钱包余额

**功能说明：**
- 系统自动判断退票时间距离航班起飞时间
- 自动计算退票手续费和退款金额
- 退款金额直接退回到用户钱包余额
- 退票成功后自动释放订单占用的座位

### 3.2 航班出发前12小时内票价上涨10% ✅

**项目要求：** 航班出发前12小时内票价上涨10%

**实现状态：** ✅ 已实现

**紧急加价规则：**
- **起飞前12小时内订票：** 在原价基础上加收 **10%** 作为紧急加价
- **起飞前12小时外订票：** 不收取紧急加价，按原价计算

**价格计算方式：**
```
最终订单金额 = 基础价格 × 票数 × (1 + 紧急加价率)
```

**功能说明：**
- 系统自动判断订票时间距离航班起飞时间
- 如果在12小时内，自动在原价基础上加收10%紧急加价
- 订单金额会自动显示含紧急加价后的总金额
- 用户可以在预订页面看到是否含紧急加价的提示

### 3.3 订票时可以添加多人信息 ✅

**项目要求：** 订票时可以添加多人信息进行订票

**实现状态：** ✅ 已实现

**多人订票规则：**
- 支持一次预订为多人购票（最多9人）
- 订票数可以设置为1-9人
- 每位乘客需要填写完整信息

**乘客信息要求：**
- **姓名：** 必填，每位乘客的姓名（可以相同，支持重名）
- **身份证号：** 必填，每位乘客的身份证号（必须唯一，不能重复）
- **联系电话：** 必填，每位乘客的联系电话（必须唯一，不能重复）

**功能特性：**
- 系统自动验证乘客信息唯一性
  - 身份证号必须唯一（不能重复使用）
  - 联系电话必须唯一（不能重复使用）
  - 姓名可以相同（支持重名情况）
- 第一位乘客默认使用当前登录用户的信息
- 其他乘客信息需要手动填写
- 系统自动为每位乘客分配座位号
- 订单金额 = 单价 × 票数（含紧急加价）

**验证规则：**
- 如果身份证号重复，系统提示"身份证号已被重复使用，请确保每位乘客身份证号唯一"
- 如果联系电话重复，系统提示"联系电话已被重复使用，请确保每位乘客联系电话唯一"
- 如果必填信息为空，系统提示相应错误信息
