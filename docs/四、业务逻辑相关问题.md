## 2. 如何处理订单状态的转换？

### 2.1 订单状态定义

**状态枚举：**
- **CREATED：** 已创建但未支付
- **PAID：** 已支付并出票
- **REFUND_REVIEW：** 退票申请审核中
- **REFUND_REJECTED：** 退票申请已驳回
- **CANCELED：** 已取消

### 2.2 状态转换流程

**正常流程：**
```
CREATED → PAID → CANCELED（退票成功）
```

**退票流程：**
```
PAID → REFUND_REVIEW → CANCELED（审核通过）
PAID → REFUND_REVIEW → REFUND_REJECTED（审核驳回）
PAID → CANCELED（自动退款）
```

### 2.3 状态转换实现

**创建订单：**
```java
booking.setStatus(payLater ? "CREATED" : "PAID");
```

**支付订单：**
```java
if ("CREATED".equals(booking.getStatus())) {
    booking.setStatus("PAID");
    // 扣减钱包余额
}
```

**申请退票：**
```java
if ("PAID".equals(booking.getStatus())) {
    if (needsReview) {
        booking.setStatus("REFUND_REVIEW");
    } else {
        booking.setStatus("CANCELED");
        // 自动退款
    }
}
```

**审核退票：**
```java
if ("REFUND_REVIEW".equals(booking.getStatus())) {
    if (approve) {
        booking.setStatus("CANCELED");
        // 执行退款
    } else {
        booking.setStatus("REFUND_REJECTED");
    }
}
```

## 3. 如何确保预订时的座位数量不超过剩余座位数？

**在创建订单前验证：**

```java
// 验证订票数不超过剩余座位数

    throw new RuntimeException("该舱位剩余座位不足");
}
```

### 3.2 剩余座位计算

**新系统（seats 表）：**
```java
private int calculateRemainingSeatsFromSeatTable(Long flightId, String cabinClass) {
    // 总可用座位数
    long availableSeats = seatRepository.countByFlightIdAndCabinClassAndIsAvailableTrue(
        flightId, cabinClass);
    
    // 已占用座位数
    long occupied = bookingSeatRepository.countOccupiedSeats(flightId, cabinClass);
    
    // 剩余座位数
    return (int) Math.max(availableSeats - occupied, 0);
}
```

## 5. 如何生成唯一的订单号？

### 5.1 订单号格式

**格式：** `BK + 日期(YYYYMMDD) + 6位序号`

**示例：** `BK20251120001234`

### 5.2 生成逻辑

**自动生成（@PrePersist）：**
```java
@PrePersist
protected void onCreate() {
    if (orderNo == null || orderNo.isEmpty()) {
        orderNo = generateOrderNo();
    }
}

private String generateOrderNo() {
    LocalDateTime now = LocalDateTime.now();
    String dateStr = String.format("%04d%02d%02d", 
        now.getYear(), now.getMonthValue(), now.getDayOfMonth());
    
    // 使用时间戳 + 纳秒时间确保唯一性
    long timestamp = System.currentTimeMillis();
    long nanoTime = System.nanoTime();
    int sequence = (int) ((timestamp + nanoTime) % 1000000);
    String sequenceStr = String.format("%06d", sequence);
    
    return "BK" + dateStr + sequenceStr;
}
```

### 5.3 唯一性保证

**数据库唯一约束：**
```sql
ALTER TABLE bookings 
ADD CONSTRAINT UK_order_no UNIQUE (order_no);
```

**唯一性检查（应用层）：**
```java
// 生成后检查是否存在（极小概率）
if (bookingRepository.existsByOrderNo(orderNo)) {
    // 重新生成
    orderNo = generateOrderNo();
}
```
